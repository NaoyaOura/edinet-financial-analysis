# 機能設計書

## 1. 研究目的

有価証券報告書における生成AI・DX関連キーワードの出現頻度と、日本企業の財務パフォーマンスとの関係を定量的に分析する。
メインの分析対象は小売業（「生成AIの活用が小売業の成長に寄与するか」の検証）とし、比較対象としてIT企業（情報通信業）も分析する。
IT企業は生成AI・DXとの親和性が高く、小売業との比較を通じて「キーワードスコアと業績の関係が業種によって異なるか」を検証する。

---

## 2. 研究仮説

| 仮説 | 内容 |
|---|---|
| H1（主仮説） | 生成AI・DX関連キーワードスコアが高い企業は、翌年の売上高成長率・営業利益率が高い |
| H2 | キーワードスコアの増加率が高い企業は、業績改善が見られる |
| H3（対立仮説） | キーワードスコアと業績に有意な関係はない（過剰なバズワード） |
| H4（業種比較） | キーワードスコアと業績の関係はIT企業において小売業よりも強く現れる |

---

## 3. 分析対象

### 3-1. 対象企業

- **国内企業のみ**（海外企業は対象外）
- **業種A（メイン）**: 東証33業種分類コード `6100`（小売業）
  - 各種商品小売業（コンビニ・百貨店・スーパー等）
  - 織物・衣服・身の回り品小売業
  - 飲食料品小売業
  - 機械器具小売業（家電・車等）
  - その他の小売業
- **業種B（比較対象）**: 東証33業種分類コード `5250`（情報・通信業 / IT企業）
  - ソフトウェア業、情報処理サービス業、インターネット付随サービス業 等
  - **目的**: 生成AI・DXキーワードスコアが小売業の業績と有意な関係を示さない場合でも、IT企業の業績には強く影響するかを業種横断で比較検証するため

> 東証33業種コードの一覧: [J-Quants API ドキュメント](https://jpx-jquants.com/ja/spec/eq-master/sector33code)
- **上場区分**: 金融商品取引法に基づきEDINETへ有価証券報告書を提出している企業
- **除外条件**: 分析期間中に上場廃止・業種変更した企業

### 3-2. 分析期間

| 期間 | 年度 |
|---|---|
| **統一分析期間** | **2022〜2025年度（4年間）** |

- 2022年度を起点とすることで、コロナ禍（2020〜2021年度）の直接的な業績影響を除外できる
- 2025年度まで含めることで、ChatGPT普及後の変化を4年間の時系列で追跡できる
- 分析期間はすべてコロナ後回復期〜生成AI普及期にあたり、コロナ禍ダミー変数は不要

## 4. データソース

### 4-1. EDINET API（テキストデータ）

- **エンドポイント**: `https://disclosure.edinet-fsa.go.jp/api/v2/`
- **取得書類**: 有価証券報告書（`docTypeCode=120`）
- **取得内容**: 報告書全文テキスト（キーワードスコア算出専用）

財務数値データはEDINET XBRLからは取得しない。財務データはすべてJ-Quantsから取得する。

### 4-2. J-Quants API（財務データ・業種分類）

- **エンドポイント**: `https://api.jquants.com/v1`
- **利用エンドポイント**:
  - `/listed/info` — 業種コード（Sector33Code）の取得
  - `/fins/statements` — 財務諸表データ（**財務データの一次ソース**）
  - `/prices/daily_quotes` — 日次株価（追加変数として利用）

### 4-3. 取得する財務データ項目（J-Quants `/fins/statements`）

| 指標名 | J-Quantsフィールド名 | 用途分類 |
|---|---|---|
| 売上高 | `NetSales` | 成長性・規模コントロール |
| 営業利益 | `OperatingProfit` | 収益性（目的変数候補） |
| 経常利益 | `OrdinaryProfit` | 収益性 |
| 当期純利益 | `Profit` | 収益性（目的変数候補） |
| 総資産 | `TotalAssets` | 規模コントロール・ROA計算 |
| 純資産 | `Equity` | 安全性・ROE計算 |
| 営業キャッシュフロー | `CashFlowsFromOperatingActivities` | キャッシュ創出力 |
| 投資キャッシュフロー | `CashFlowsFromInvestingActivities` | 投資活動 |
| 財務キャッシュフロー | `CashFlowsFromFinancingActivities` | 資金調達活動 |
| 現金及び現金同等物 | `CashAndEquivalents` | 流動性 |

### 4-4. テキスト分析対象セクション

有価証券報告書の以下のセクションからテキストを抽出：

- **第2 事業の状況 > 1. 経営方針、経営環境及び対処すべき課題等**
- **第2 事業の状況 > 2. サステナビリティに関する考え方及び取組**
- **第2 事業の状況 > 4. 経営者による財政状態、経営成績及びキャッシュ・フローの状況の分析（MD&A）**

選定根拠・除外理由 → [text-section-selection.md](text-section-selection.md)

## 5. キーワード定義

### 5-1. キーワードカテゴリ

| カテゴリ | キーワード例 | 重み |
|---|---|---|
| 生成AI（強） | 生成AI、生成人工知能、ChatGPT、GPT-4、LLM、大規模言語モデル | 3 |
| AI全般（中） | AI、人工知能、機械学習、ディープラーニング、自然言語処理 | 2 |
| DX（弱） | DX、デジタルトランスフォーメーション、デジタル変革、デジタル化 | 1 |

### 5-2. キーワードスコア算出

```
キーワード密度 = Σ(出現回数 × 重み) ÷ 総文字数 × 10,000
```

- 総文字数で正規化することで、報告書の長さによるバイアスを排除
- 否定文脈（「導入していない」「検討していない」等）の前後N文字は除外

### 5-3. 企業グルーピング

各年度のキーワード密度を四分位で分類：

- **高スコア群（Q4）**: 上位25%
- **中スコア群（Q2-Q3）**: 中間50%
- **低スコア群（Q1）**: 下位25%

## 6. 分析手法

### 6-1. 基本分析：グループ比較

高・中・低スコア群の財務指標平均値を比較し、有意差を検定（t検定・ANOVA）。
比較する指標は以下の複数パターンを検証する（「6-5. 変数探索」参照）。

### 6-2. ラグ回帰分析（メイン分析）

翌年の業績変化を目的変数、当年のキーワードスコアを説明変数とした回帰分析。

```
業績指標(t+1) = β₀ + β₁×キーワードスコア(t) + β₂×コントロール変数群(t) + ε
```

- 1年ラグ・2年ラグの両方で実施
- コントロール変数は変数探索によって最適な組み合わせを選定

### 6-3. 差分の差分法（DiD）

2023年を「処理期間」の開始として、キーワードが急増した企業群と増えなかった企業群を比較。

```
処理群：2023年度にキーワードスコアが前年比50%以上増加した企業
対照群：同期間にキーワードスコアの変化が軽微な企業
```

### 6-4. パネルデータ分析

固定効果モデルで企業固有の不変要因を除去した上で分析。

### 6-5. 変数探索（多仕様比較）

分析に使用する変数は固定せず、以下の候補から組み合わせを体系的に検証して最適仕様を特定する。

#### 目的変数（被説明変数）の候補

| 変数名 | 定義 | 分類 |
|---|---|---|
| 営業利益率 (%) | operatingProfit / netSales × 100 | 収益性 |
| ROA (%) | profit / totalAssets × 100 | 収益性 |
| ROE (%) | profit / equity × 100 | 収益性 |
| 純利益率 (%) | profit / netSales × 100 | 収益性 |
| 自己資本比率 (%) | equity / totalAssets × 100 | 安全性 |

#### キーワード変数（主説明変数）の候補

| 変数名 | 内容 |
|---|---|
| totalScore | 全カテゴリの重み付きキーワード密度 |
| genAiScore | 生成AI関連キーワード（重み3）のスコア |
| aiScore | AI全般キーワード（重み2）のスコア |
| dxScore | DXキーワード（重み1）のスコア |

#### コントロール変数の候補

| 変数名 | 定義 |
|---|---|
| log(売上高) | ln(netSales) — 企業規模コントロール |
| log(総資産) | ln(totalAssets) — 企業規模コントロール（代替） |
| 自己資本比率 | equity / totalAssets — 財務健全性（目的変数でない場合） |
| 小売業ダミー | Sector33Code = "6100" なら 1（参照グループ: IT業・その他） |

> **注意**: 小売業ダミーと情報通信業ダミーを同時投入するとダミー変数トラップ（完全多重共線性）が生じるため、片方のみ投入する。本プロジェクトでは小売業ダミーを採用する。

#### 変数選択の評価基準

1. **R²・調整済みR²**: 説明力の高いモデルを優先
2. **係数の統計的有意性**: p値（*** p<.01 / ** p<.05 / * p<.10）
3. **係数の符号と方向**: キーワードスコアの係数が正（仮説H1と整合するか）
4. **多重共線性**: 相関の高い変数を同時投入しない
5. **理論的整合性**: 統計的有意性だけでなく業務上の解釈可能性

`analyze --type explore` コマンドで全組み合わせの比較表を一括出力する。

## 8. 制約・前提条件

| 項目 | 内容 |
|---|---|
| EDINET APIキー | 環境変数 `EDINET_API_KEY` から取得。ソースコードへの直接記述・コミット禁止 |
| J-Quants APIキー | 環境変数 `JQUANTS_REFRESH_TOKEN` から取得。同様にコミット禁止 |
| J-Quantsプラン | Standard プラン（Free プランでは取得期間・件数に制限あり） |
| 財務データ | J-Quants `/fins/statements` を一次ソースとして利用 |
| テキストデータ | EDINET 有価証券報告書（日本語テキストのみ対象） |
| XBRL解析 | 財務データの取得にはXBRL解析を使用しない |
| 業種分類 | J-Quants Sector33Code を正規データとして利用 |
| 統計ライブラリ | Apache Commons Math |
| 除外業種 | 分析対象は小売業（6100）・情報通信業（5250）。ただし探索的な変数検証時はこの制限を緩和可能 |
